# 23 Interface Interview Q&A

## Concept Explanation
This topic focuses on common interview questions about interfaces in C#, such as purpose, benefits, limitations, and differences from abstract classes.

## Real-World Analogy
A contract describes what services must be delivered, while each company decides how to provide them. That is how interfaces work in software design.

## Syntax
```csharp
public interface IPrinter
{
    void Print();
}
```

## Step-by-Step Explanation
1. Understand what an interface is.
2. Learn why interfaces support flexibility.
3. Compare interfaces with abstract classes.
4. Practice concise interview-ready answers.

## Best Practices
- Explain interfaces using contracts and flexibility.
- Use simple examples in interviews.
- Compare abstract classes and interfaces clearly.

## Common Mistakes
- Saying interfaces contain full implementation in every case.
- Ignoring why multiple interface implementation matters.
