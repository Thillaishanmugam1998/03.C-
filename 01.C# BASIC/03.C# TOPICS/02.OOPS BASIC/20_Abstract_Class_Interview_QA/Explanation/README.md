# 20 Abstract Class Interview Q&A

## Concept Explanation
This topic focuses on practical interview-style understanding of abstract classes in C#. Abstract classes are commonly asked in interviews because they combine inheritance, abstraction, and design decisions.

## Real-World Analogy
A company policy document defines common rules for all branches, but each branch follows those rules in its own way. That is how abstract classes guide derived classes.

## Syntax
```csharp
public abstract class Report
{
    public abstract void Generate();
}
```

## Step-by-Step Explanation
1. Understand what an abstract class is.
2. Know why it cannot be instantiated.
3. Understand abstract vs non-abstract members.
4. Learn when abstract classes are better than interfaces.
5. Practice with short question-and-answer examples.

## Best Practices
- Learn both concept and use case.
- Practice explaining abstract classes in simple words.
- Compare them with interfaces clearly.
- Use examples from real applications.

## Common Mistakes
- Saying abstract classes are the same as interfaces.
- Forgetting that abstract classes can contain implemented methods.
- Missing the purpose of shared state in abstract classes.
